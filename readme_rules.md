В репозитории два уровня README, оба должны всегда быть актуальными.

## Уровень 1 — README отдела

Файл: `<отдел>/README.md`

Содержит:
- Одна строка: что делает отдел
- Таблица всех скиллов: название (ссылка) + одно предложение что делает
- Файловая структура отдела

**Когда обновлять:** при добавлении, удалении или переименовании скилла в этом отделе.

## Уровень 2 — README скилла

Файл: `<отдел>/skills/<skill-name>/README.md`

Содержит:
- Заголовок = название скилла
- Что это (1–2 предложения)
- Как работает (нумерованные шаги)
- Входные данные (таблица или список)
- Выходные данные (таблица или список)
- Триггеры (фразы и ситуации)
- Файловая структура (дерево)

**Когда обновлять:** при изменении алгоритма, добавлении/удалении файлов или изменении входных/выходных данных скилла.

**Why:** пользователь хочет поддерживать актуальность всех README автоматически при работе с репозиторием.
**How to apply:** каждый раз при создании нового скилла или изменении существующего — обновлять оба README (скилла и отдела). Не забывать про README отдела при добавлении первого скилла в пустой отдел.
