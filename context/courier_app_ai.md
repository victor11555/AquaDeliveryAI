# Курьерское приложение Aqua Delivery — справочник для ИИ-агента

> **Что это такое:** мобильное приложение для водителей-курьеров Aqua Delivery.
> Курьер видит свои заказы на день, строит маршрут, звонит клиентам, редактирует состав заказа, сканирует маркировку, фиксирует оплату и закрывает смену через отчёт за день.
>
> **Версия:** 2.9.5

## Как устроена работа курьера — общая логика

```
Курьер открывает приложение
  → Список заказов (карта или список, сгруппировано по интервалам)
  → Карточка заказа (адрес, клиент, товары, оплата)
      → Уведомить клиента «Буду через 10/20/30 мин»
      → Позвонить клиенту
      → Редактировать состав (добавить/убрать товар)
      → Построить маршрут (Яндекс Карты или другое)
  → Завершение:
      «Доставлен» → указать оплату → подтвердить
      «Не доставлен» → написать причину → подтвердить
  → Отчёт за день (статистика + суммы по способам оплаты + тара)
```

**Ключевые принципы:**
- Заказы группируются по временным интервалам; просроченные выделяются красной полосой.
- Курьер может редактировать состав заказа прямо в приложении — но если в заказе есть маркировка, при изменении состава коды маркировки сбрасываются и нужно сканировать заново.
- Статус «Работаю» (тумблер в меню) управляет доступностью курьера в системе.
- Есть офлайн-режим — работа без синхронизации с сервером.
- Маршрут строится от точки старта, которую курьер задаёт вручную или через автоопределение.

## 1. Список заказов

### 1.1 Режимы отображения

| Режим | Описание |
|---|---|
| **Карта** | Заказы на карте города с кластеризацией. Числа на маркерах = кол-во заказов в кластере. Фильтр по интервалу — кнопка с временем в правом верхнем углу (пример: «8:00–22:00») |
| **Список** | Вертикальный список заказов, сгруппированных по временным интервалам |

### 1.2 Группировка и статусы

| Статус / тип | Описание |
|---|---|
| **Просрочен (красная полоса)** | Заказ с истёкшим интервалом. Показывается дата/время интервала и счётчик заказов в группе |
| **Сегодня (зелёная полоса)** | Заказы на текущий день. Показывается интервал и счётчик |

### 1.3 Карточка заказа в списке

| Элемент | Описание |
|---|---|
| Номер и имя клиента | № заказа + имя/название в шапке карточки |
| Зона и город | Зона доставки / город (пример: «Сыктывкар и пригород / Сыктывкар») |
| Адрес | Улица и номер дома |
| Сумма заказа | Итоговая сумма (₽), правый верхний угол |
| Иконка бутылки + число | Количество бутылей в заказе |
| Иконка «+» + число | Количество дополнительных (небутылочных) товаров |
| Иконка QR (красная) | Заказ требует сканирования кодов маркировки |
| Иконка оплаты (монеты) | Способ оплаты: монеты = наличные/эквайринг; Wi-Fi = онлайн-оплата |
| Кнопка «Маршрут» | Открывает экран построения маршрута до адреса |

## 2. Карточка заказа

### 2.1 Основная информация

| Элемент | Описание |
|---|---|
| Адрес доставки | Полный адрес с зоной/городом. Кнопка «стрелка» рядом → навигация в Яндекс Картах (или другом приложении) |
| Дата и время | Дата, день недели и интервал (пример: «27 ноября 2024, ср, 13:00–13:10») |
| Способ оплаты | Ссылка-кнопка: QR code / Юмани без привязки / Юмани с привязкой / Юкасса с привязкой / Наличные / По договору и др. |
| Задолженность по таре (розовая плашка) | Показывается, если у клиента есть долг по возвратной таре (пример: «Задолженность — 226 бутылей») |

### 2.2 Уведомление клиента «Буду через»

| Кнопка | Действие |
|---|---|
| «10 мин.» | Отправляет клиенту push — курьер прибудет через 10 минут |
| «20 мин.» | Через 20 минут |
| «30 мин.» | Через 30 минут |

### 2.3 Данные клиента

| Элемент | Описание |
|---|---|
| Имя и номер клиента | ФИО или название компании + внутренний номер клиента в системе |
| Телефон | Кнопка с номером — нажатие инициирует звонок через стандартный телефон устройства |

### 2.4 Список товаров

| Элемент | Описание |
|---|---|
| Заголовок «Товаров: N» | Общее кол-во позиций в заказе |
| Иконка редактирования | Открывает экран редактирования состава (добавление / изменение количества) |
| Строка товара | Название, количество (шт.) и цена за позицию |
| Набор | Комплект товаров — отдельная строка с суммой набора |
| Доставка | Стоимость доставки по тарифу зоны |
| Итого / К оплате | Итоговая сумма к получению с клиента |

### 2.5 Действия с заказом

| Кнопка | Что открывает |
|---|---|
| **«Доставлен»** (синяя) | Экран подтверждения доставки: поле комментария + статус оплаты + кнопка QR для приёма оплаты + кнопка «Подтвердить» |
| **«Не доставлен»** (розовая) | Экран с обязательным полем «Напишите причину» + кнопка «Подтвердить» |

## 3. Завершение заказа

### 3.1 Экран «Заказ доставлен»

| Элемент | Описание |
|---|---|
| Смайл 😊 | Визуальное подтверждение успешной доставки |
| Поле комментария | Необязательный комментарий к выполненному заказу |
| Блок оплаты | Способ оплаты + статус: «не оплачено» (красный) или «оплачено» |
| Кнопка QR-кода | Открывает сканер / генератор QR для приёма оплаты по динамическому QR-коду |
| **«Подтвердить»** | Фиксирует заказ как доставленный. При нажатии — всплывает диалог: «Вы собираетесь принять оплату по QR-коду. Изменить способ оплаты?» |

### 3.2 Экран «Заказ не доставлен»

| Элемент | Описание |
|---|---|
| Смайл 😞 | Визуальное отображение недоставки |
| Поле «Напишите причину» | **Обязательное** текстовое поле — причина недоставки |
| **«Подтвердить»** | Фиксирует заказ как недоставленный с указанной причиной |

## 4. Редактирование состава заказа

### 4.1 Изменение количества

| Элемент | Описание |
|---|---|
| Кнопка «–» | Уменьшить кол-во позиции на 1 |
| Счётчик (число) | Текущее количество единиц |
| Кнопка «+» | Увеличить кол-во на 1 |

### 4.2 Добавление товаров

| Элемент | Описание |
|---|---|
| Кнопка «Добавить товар» | Открывает экран каталога |
| Вкладки категорий | Вода / Услуги / Оборудование / Акции / Напитки — горизонтальная прокручиваемая лента |
| Карточка товара | Фото, название, цена. Кнопка «+» — добавить в заказ |
| Кнопка «Добавить» | Подтвердить добавление выбранных товаров |
| Кнопка «Отменить» | Закрыть без сохранения |

### 4.3 ⚠️ Предупреждение о маркировке

При изменении состава заказа, в котором уже есть сохранённые коды маркировки, появляется диалог:

> «В заказе есть сохранённые коды маркировок. При изменении состава заказа коды маркировок будут удалены и для завершения заказа потребуется заново отсканировать маркировки. Хотите продолжить?»

Кнопки: **«Нет»** (отмена) / **«Да»** (продолжить и сбросить коды).

### 4.4 Сохранение изменений

| Элемент | Описание |
|---|---|
| Кнопка «Сохранить» | Сохраняет изменения. Показывает зелёный баннер: **«Заказ успешно изменён»** |
| Кнопка «Отмена» | Закрывает экран без сохранения |

## 5. Маршрутизация

### 5.1 Режим карты (главный экран)

| Элемент | Описание |
|---|---|
| Маркеры заказов | Числа на карте = кластеры / отдельные заказы. Клик → маршрутный лист для этой группы |
| Фильтр по интервалу | Кнопка с интервалом в правом верхнем углу. Клик → выпадающий список всех доступных интервалов |
| Кнопка изменения маршрута | Открывает маршрутный лист с возможностью перестановки заказов |
| Кнопка геолокации | Открывает форму изменения точки старта маршрута |
| Кнопка внешней карты | Открывает маршрут во внешнем картографическом приложении |
| Синяя точка | Текущее положение курьера на карте |

### 5.2 Маршрутный лист

| Элемент | Описание |
|---|---|
| Список адресов | Пронумерованный список адресов в рамках выбранного интервала |
| Иконка перетаскивания (≡) | Изменить порядок заказов методом drag & drop |
| Кнопка «Сохранить маршрут» | Сохраняет новый порядок. Показывает баннер: **«Порядок заказов был успешно изменён»** |
| Кнопка «Отмена» | Сбросить изменения без сохранения |

### 5.3 Точка старта маршрута

| Элемент | Описание |
|---|---|
| Карта с маркером | Интерактивная карта — маркер перетаскивается для выбора точки старта |
| Поле «Широта» | Числовой ввод (пример: 61.660764) |
| Поле «Долгота» | Числовой ввод (пример: 50.825827) |
| «Включить автоопределение местоположения» | Подставляет текущие GPS-координаты устройства. Если разрешение не дано — диалог с кнопкой «В настройки» |
| «Сохранить» | Сохраняет точку старта |
| «Отменить» | Закрыть без сохранения |

### 5.4 Построение маршрута к заказу

| Элемент | Описание |
|---|---|
| Экран со списком адресов | Адрес заказа (или несколько) с номером заказа |
| «Построить маршрут» | Строит маршрут от точки старта до адреса(ов) и отображает на карте |
| Обязательный выбор времени старта | Если время не выбрано — диалог «Необходимо выбрать время начала маршрута» с кнопкой «Продолжить» |
| Выбор даты | День + месяц + год или час + минута |

## 6. Геолокация

| Ситуация | Поведение |
|---|---|
| Доступ не предоставлен | Диалог: «Доступ к данным о местоположении. Разрешите в настройках телефона приложению «Курьер» доступ к данным о местоположении устройства» + кнопка «В настройки» |
| Автоопределение включено | Синяя иконка стрелки в статусной строке = геолокация активна. Синяя точка на карте = текущее положение курьера |

## 7. Отчёт за день

### 7.1 Навигация по датам

| Элемент | Описание |
|---|---|
| Строка дней недели | Горизонтальная полоса с датами. Текущая дата выделена белым прямоугольником |
| Стрелки «‹» / «›» | Переключение недели назад / вперёд |
| Иконка календаря + месяц | Открывает полноэкранный датапикер (число + месяц + год) для перехода к любой дате |
| Кнопка «Все маршруты» | Фильтр по маршруту. Открывает список интервалов для выбора конкретного маршрута |

### 7.2 Статистика заказов

| Показатель | Описание |
|---|---|
| Заказы (всего) | Общее кол-во заказов за выбранный день |
| Доставлено | Успешно доставленных |
| Новый / Принят / Выполняется | Заказы в активных статусах (ещё не завершены) |
| Не доставлено | Отмеченных как недоставленные |
| Отменено | Отменённых заказов |

### 7.3 Общая сумма оплат (9 способов)

| Способ оплаты | Описание |
|---|---|
| Наличные | Принято наличными. Кнопка «›» → детализация по заказам |
| Онлайн-оплата | QR, СБП, Юмани и др. |
| По договору | Для юридических лиц |
| Картой | Банковской картой |
| По телефону | Через звонок / по телефону |
| Динамический QR-код | При доставке через QR |
| Купонами | Промокоды / купоны |
| ЕРИП | Белорусская система платежей |
| Баллами | Бонусные баллы клиента |

### 7.4 Возвратная тара

| Показатель | Описание |
|---|---|
| План | Плановое кол-во бутылей к возврату за день |
| Возвращено | Фактически возвращено (прокрутка вниз для просмотра) |

## 8. Боковое меню

| Элемент | Описание |
|---|---|
| Имя и телефон | Имя профиля курьера и номер телефона в шапке меню |
| «Выйти» (синяя ссылка) | Выход из учётной записи |
| Заказы | Переход к главному экрану (список / карта активных заказов) |
| Выполненные заказы | Список завершённых заказов смены; при отсутствии — заглушка «Тут пока пусто 🙁» |
| Отчёт за день | Переход к экрану ежедневного отчёта |
| Тумблер **«Работаю»** | Вкл/выкл статус доступности. Зелёный = курьер активен в системе |
| Тумблер **«Офлайн режим»** | Включает работу без синхронизации с сервером |
| Версия приложения | Текущая версия (пример: 2.9.5) |

## 9. Выполненные заказы

| Состояние | Описание |
|---|---|

| Пустой экран | Иконка облака + надпись «Тут пока пусто 🙁» |
