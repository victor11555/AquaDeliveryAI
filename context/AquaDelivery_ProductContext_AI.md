## 1. Что такое Aqua Delivery

**Aqua Delivery** — SaaS B2B платформа для автоматизации бизнеса по доставке воды.

**Бизнес-модель:** продаём платформу компаниям-партнёрам (службам доставки воды), которые используют её для обслуживания своих клиентов-покупателей. White-label: каждый партнёр получает брендированную версию продуктов под свои цвета и логотип.

**Масштаб (май 2026):**
- 422 партнёра, из них 245 активных за последние 30 дней
- 910 071 уникальных конечных клиентов
- ~1 млн заказов за 90 дней
- Рейтинг клиентского приложения: 4.33 (62 849 отзывов)
- Присутствие: Россия, Казахстан, Украина

## 2. Бизнес доставки воды — как устроен рынок

### Продуктовая линейка компаний-партнёров

| Категория | Особенности |
|---|---|
| Вода 19л в многооборотной таре (ПЭТ/ПК) | **Основной продукт.** Клиент выкупает тару при первой покупке, затем возвращает пустую и получает полную. Тара ПК выдерживает ~40 циклов, ПЭТ ~20 (данные приблизительные, зависят от поставщика). Схемы выкупа тары: полный выкуп 100% / тара под залог (клиент может в любой момент потребовать возврат) / бесплатная аренда (с условиями договора, при нарушении — списание с карты). Схемы: полный выкуп / залог / бесплатная аренда |
| Вода 1.5л / 0.5л в одноразовой таре | В основном для юрлиц (HoReCa, производства). Упаковки по 6/12/18/24 шт |
| Помпы механические/электрические | Продаётся 1 раз, редко меняется |
| Кулеры напольные/настольные | 99% — юрлица (офисы, школы). Полный выкуп или залог |
| Наборы одноразовых стаканчиков | По 100 шт, преимущественно юрлица |
| Детская питьевая вода 5л/1.5л/0.5л | Особые требования по минеральному составу |
| Лимонады 1.5л | В упаковках по 6 шт |
| Набор «вода+тара+помпа» (комбо) | Для новых клиентов по сниженной цене — конвертер |

**Учёт тары — ключевая сложность бизнеса:** нужно мониторить у кого сколько тары, кто сколько должен. Если партнёр не является производителем — ещё усложняется учёт тары с поставщиками.

### Типы компаний на рынке

- ~2000 компаний в РФ, занимающихся доставкой воды 19л
- По обороту: микро (<3 млн/мес, 25%) → малые-средние (3–5 млн, 30%) → средние (5–8 млн, 25%) → крупные-средние (8–12 млн, 15%) → крупные (>12 млн, 5%)
- По производству: имеют своё производство + розница / только дистрибьютор
- По источнику воды: скважина vs. водопровод + многоступенчатая очистка + минерализация

### Конкуренты на рынке IT для доставки воды

Водавоз, Своя Система, Водяной, WaterApp — и Aqua Delivery (220+ партнёров на момент написания).

### Логистика — как работает

- **Логист** получает заказы, сгруппированные по районам/интервалам, распределяет по водителям с учётом грузоподъёмности машин и удалённости адресов. Формирует **маршрутные листы** (МЛ).
- **Курьер** получает МЛ, последовательно развозит. Топовые курьеры — до 3000 бутылей/месяц (~60 тонн ручной работы). Топовые курьеры доставляют до 3000 бутылей/месяц (~60 тонн ручной работы). Зарплата: фиксированная ставка за бутыль 19л + % от выручки других товаров. Главная проблема — поиск и отток курьеров (конкурируют Яндекс Лавка, Самокат, Яндекс Такси).
- **Основные расходы:** зарплаты логиста/курьеров/экспедиторов, ГСМ, лизинг машин.
- Типичная модель ценообразования: от 2 бутылей — снижение цены за штуку (курьер несёт одинаковые затраты).
- Маркетинг: LTV очень высокий — повторные продажи составляют до 98% выручки, клиент может годами заказывать у одной компании.

## 4. CRM «Кабинет заказов»

**Доступ:** `v.домен-компании.aqua-delivery.ru`  
**Что это:** веб-приложение для операторов колл-центра и логистов партнёра. Основной рабочий инструмент для приёма и обработки заказов.

### Технический стек

- **Framework:** React 18 + TypeScript
- **Build:** Rspack
- **State:** Redux / Zustand / Context API
- **HTTP:** Axios
- **Routing:** React Router v6
- **Структура репо:** `src/components/`, `src/pages/`, `src/hooks/`, `src/services/`, `src/store/`

### Основные модули

**Заказы**
- Два режима: канбан и таблица
- Статусы: Новый → Принятый → Выполняется → Доставлено / Отменён / Не доставлено
- Создание вручную (по звонку): клиент, адрес, дата/время, товары, способ оплаты
- Карточка: редактирование состава/водителя/адреса/оплаты, комментарии оператора
- Экспорт CSV / Excel

**Клиенты**
- Поиск по ФИО, телефону, email, ID
- Карточка: история заказов, долг по таре, баллы, контакты, индивидуальные цены

**Маршрутные листы**
- Создание через фильтры (район, курьер, дата, статус)
- Добавление заказов в маршрут, печать, удаление
- Назначение координат адресов вручную

**Водители**
- Список, фильтр по активным
- Карточка: история, отчёт по эффективности, экспорт

**Коммуникации**
- Встроенный чат с клиентами (без мессенджеров)
- Обратная связь: обращения, негативные отзывы
- Пропущенные звонки (IP-телефония)

**Система лояльности**
- Автоматическое и ручное начисление/списание баллов
- История, экспорт, промокоды

**Возврат клиентов**
- «Отвалившиеся» — по умолчанию нет заказов 60+ дней (параметр настраивается в Кабинете настроек)
- Каналы: Push, Email, SMS, звонок

**Возвратная тара**
- Список должников, фильтр по зонам, история операций, ручные записи

**Планировщик заказов**
- Регулярные заказы: клиент + шаблон + день недели + время

**Онлайн-оплаты**
- Отчёт по платежам через приложение и сайт
- Дневной отчёт: кол-во заказов, статусы, способы оплаты

**Воронки**
- Канбан-доски для анализа клиентской базы (новые / активные / под риском / спящие / потерянные)
- Drag & Drop между этапами воронки возврата

### Кабинет настроек (административная панель)

**Товары:** категории, товары (цена, остатки, изображения, зоны, сопутствующие, мин/макс кол-во, тип товара), массовое изменение цен.

**Баннеры и акции:** 4 типа — информационный / внешняя ссылка / история / набор товаров. Периоды, зоны, лимит заказов для отмены акции.

**Пользователи:** клиенты (импорт/экспорт, инд. цены), водители, операторы, маркетологи, администраторы.

**Push-уведомления:** заголовок (макс. 125 символов), текст, получатели (конкретные / все), изображение, страница в приложении, внешняя ссылка, расписание.

**Система лояльности:** % кешбэка, инвайты (+баллы за приглашение), промокоды (% скидка + дата), причины начислений, баллы за регистрацию, настройки десятичных баллов.

**География:**
- Зоны доставки (координаты полигонов, город по умолчанию, офис, ассортимент по зоне)
- Районы (карта, приоритет, активность)
- Интервалы доставки (день недели, время с/до, лимит заказов, скрытие за N минут, инд. цена доставки, флаг «закрытие с конца»)
- Стоимость доставки по районам (тариф + порог бесплатной)
- Нерабочие дни (период + привязка к районам)

**Настройки кабинета заказов:**
- Расписание операторов (по дням недели)
- Автоответ в нерабочее время
- Рабочее время для ЭКСПРЕСС-заказов
- Push-уведомления: незарегистрированным / без заказов / «отвалившимся»
- Статус «Требует подтверждения» — условия: стоп-отгрузка / первый заказ / новый адрес / оплата по договору / повторный заказ на тот же день / незарегистрированный клиент / клиент организации без договора / первый заказ из приложения на организацию
- Настройки курьерского приложения (вкл/выкл): редактирование корзины, добавление товаров, телефон клиента, архив заказов (0/10/25/50/100), смена оплаты, долг и тара, маршрут, дневной отчёт, накладная на email, поле «Наличных принято», авто-статус «Принят» при назначении курьера

**Активность клиентов:** аналитика по устройствам (iOS/Android/web), версиям ОС и приложения, дата установки, дата последней активности.

**Дополнительно:** офисы продаж (контакты, реквизиты, соцсети, документы, сертификаты), языковые настройки, первый заказ (pop-up о возвратной таре), подсказки в чате, настройки чата, SMS-шаблоны.

## 7. Aquamarket (IoT)

**Что это:** микросервис для управления умными торговыми автоматами с водой.

**API:** `/aquamarket/*` (REST, JWT)

### Возможности

- Управление группами автоматов (BoxGroup) и отдельными ячейками (Box)
- Запросы: создание / заполнение / изменение состояния ячеек
- Сканирование маркировочных кодов при отгрузке
- Логирование в ClickHouse: `iot_log_stream`, `iot_order_log`, `iot_box_ping`
- Интеграция с `micro_market` заказами

### API-эндпоинты (сокращённо)

| Метод | Путь | Описание |
|---|---|---|
| GET/POST | `/aquamarket/boxGroup` | Коллекция групп автоматов |
| GET/PUT/DELETE | `/aquamarket/boxGroup/{id}` | Конкретная группа |
| GET/POST | `/aquamarket/box` | Коллекция ячеек |
| GET/PUT/DELETE | `/aquamarket/box/{id}` | Конкретная ячейка |
| POST | `/aquamarket/boxStateRequest` | Запрос состояния Box |
| POST | `/aquamarket/createBoxRequest` | Создание Box |
| POST | `/aquamarket/fillBoxCellsRequest` | Заполнение ячеек |
| GET/POST | `/aquamarket/location` | Локации автоматов |

> ⚠️ **Пробел:** видеозапись заказа, отключение ячеек с просроченными товарами, возврат набора, GroupOrder, локации — упомянуты в API, но без инструкций.

## 11. Бизнес-процессы

### Жизненный цикл заказа

```
Клиент оформляет заказ (приложение / сайт / звонок оператору)
    ↓
NEW → оператор принимает → ACCEPTED
    ↓
Логист включает в маршрутный лист → назначает курьера
    ↓
Курьер получает заказ в приложении → IN_PROGRESS
    ↓
Курьер доставляет → DELIVERED
(или NOT_DELIVERED если не удалось — обязательная причина)
```

### Система лояльности

- Начисление: % от стоимости заказа (настраивается по товарам)
- Курс: 1 балл = 1 рубль; +5% за каждую покупку после успешной доставки
- Срок действия: баллы могут сгорать — «Списание баллов по истечению срока давности»
- Начисляется только после доставки, за отменённые — не начисляется
- Списание: на любой товар при оформлении через приложение/сайт
- Ручное начисление оператором (подарок, компенсация)
- Инвайт-программа: +100 баллов за приглашённого, оформившего первый заказ
- Промокоды: % скидка с датой окончания

### Многоязычность

- Таблицы переводов: `product_translation`, `category_translation`
- Поддержка: RU, EN, UZ, AZ, TR, KK
- Язык по умолчанию настраивается в кабинете партнёра

## 14. Известные ограничения и технический долг

| Проблема | Детали |
|---|---|
| GPS-трек курьеров | Только точечные события (`order_delivered`, `markings_saved`). **Непрерывной трансляции нет.** Для «курьер на карте» нужно строить стрим с нуля |
| Монолит Yii2 | Основной бэкенд не разбит на микросервисы. Вынесены отдельно только: платежи, авторизация, партнёры, телефония |
| Адреса без координат | 1.4% заказов (≈14К за 90 дней) создаётся без lat/lng. Нужен fallback-геокодинг |
| Версии React Native расходятся | Курьерское на RN 0.72, клиентское на RN 0.79.6. При обновлениях — синхронизировать |
| Отсутствует подключение эквайринга в доках | Нет пошаговой инструкции для партнёра по подключению Payment Service |
| PWA не задокументировано | Есть как продукт, но нет инструкции: как включить, настроить иконку, APK, рекламный фид |
| Telegram бот-аналитик | 8 типов уведомлений в реальном времени, но нигде не задокументирован |
| VEEL.pro | Отдельное приложение для кладовщика без каких-либо инструкций |
| Data Lens | Аналитические дашборды упомянуты на сайте, нигде не задокументированы |

## 16. Идеи и дорожная карта

### Высокий impact

| Идея | Категория | Сложность |
|---|---|---|
| Предиктивный заказ воды (AI анализирует историю → «пора заказать» + готовая корзина) | AI | Средняя |
| IoT-датчик уровня воды в кулере (авто-заказ при падении остатка) | IoT | Высокая |
| Публичный API для корпоративных клиентов (подключение ERP/1С) | B2B | Высокая |
| Подписка с фиксированной ценой (рекуррентный платёж, счётчик доставок) | UX | Средняя |
| Telegram Mini App (полноценный заказ внутри Telegram) | UX | Средняя |
| Экспресс-доставка «за 2 часа» (отдельная очередь в CRM, приоритет) | UX | Средняя |
| Тепловая карта спроса по городу (ClickHouse + Mapbox/Яндекс) | Аналитика | Средняя |
| Интеграция с маркетплейсами (Ozon, Яндекс Маркет, WB) | Интеграции | Высокая |
| White-label платформа для рынков СНГ | Интеграции | Высокая |

### Средний impact

| Идея | Категория | Сложность |
|---|---|---|
| Голосовой заказ через Яндекс Алису | AI | Средняя |
| NLP-анализ отзывов (авто-задачи в CRM) | AI | Средняя |
| NFC-метка на кулере (подношение → открывает корзину) | IoT | Низкая |
| Партнёрство с застройщиками (промокод при заселении) | B2B | Низкая |
| Виджет заказа для встройки на сайт партнёра (iframe/PWA) | UX | Средняя |
| «Социальный заказ» с соседями (групповой заказ + скидка) | UX | Средняя |
| Рейтинг и KPI водителей с мотивацией (ClickHouse + дашборд) | Аналитика | Низкая |

### Быстрые идеи (низкая сложность)

1. **Подписка и автозаказ** — настроить «каждую пятницу 2 бутыли» → заказы создаются автоматически
2. **Live-трекинг курьера** — GPS в реальном времени (-40–50% звонков «когда привезёте?»)
3. **Авто-реактивация спящих клиентов** — 30+ дней → авто-SMS/push с промокодом, возвращает 15–25%
4. **Уровни лояльности** — разный % кешбэка по накопленному обороту, прогресс-бар до следующего уровня
5. **Уведомления в WhatsApp / Telegram** — вместо SMS, дешевле и удобнее
6. **RFM-сегментация в кабинете** — новые / активные / под угрозой / ушедшие
7. **Голосовой бот приёма заказов** — IVR/LLM без оператора 24/7
8. **LLM чат-бот** — первый уровень поддержки, снижает нагрузку на операторов

### Технические улучшения

- Переход на **GraphQL BFF** для фронтендов
- Event-driven архитектура на **Kafka** (вместо старых очередей)
- Полная **OpenAPI** документация + генерация клиентов
- A/B-тестирование + **feature flags**
- Обновление зависимостей: Node.js 20.x, RN 0.75+, TypeScript 5.x
