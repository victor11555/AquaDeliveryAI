# Support — Среда отдела

Техническая поддержка партнёров. Скиллы анализируют коммуникацию, синхронизируют базу знаний и создают задачи в трекере.

---

## Скиллы

| Скилл | Как вызвать | Триггеры |
|-------|-------------|----------|
| `instruction-sync` | `/instruction-sync` | «синхронизируй инструкцию», «обнови инструкцию», «сделай PR» |
| `partner-dialog-analyzer` | `/partner-dialog-analyzer` | «проанализируй диалог», «проверь качество», «посмотри переписку» |
| `partner-reply` | `/partner-reply` | «напиши ответ партнёру», «оформи как ответ», черновые заметки оператора |
| `partner-appeals-analyzer` | `/partner-appeals-analyzer` | «разбери обращения», «топ проблем по 1С», «на что писать инструкции» |
| `yandex-tracker-issue-creator` | `/yandex-tracker-issue-creator` | «создай задачу», «занеси в трекер», «поставь таску» — общий скилл из `shared/` |

---

## Что делает каждый скилл

**instruction-sync** — синхронизирует инструкции из документов пользователя с GitHub-репозиторием `victor11555/aqua-delivery-instructions`. Проверяет актуальность, диагностирует проблемы, создаёт Pull Request с обновлённой инструкцией.

**partner-dialog-analyzer** — анализирует диалоги с партнёрами (чаты и транскрибации звонков) по TOV и задачам. Оценивает качество коммуникации поддержки и выявляет потерянные договорённости. Генерирует Word-отчёт.

**partner-reply** — превращает черновые заметки оператора («ошибка в логах, фиксим, ~час, партнёр Михаил») в готовое сообщение партнёру по стандартам TOV Aqua Delivery. Применяет 5 принципов голоса: конкретика, дружелюбно, уверенно, берёт ответственность, доводит до конца.

**partner-appeals-analyzer** — читает архив обращений партнёров (Excel + Markdown), собирает единую таблицу, группирует по категориям, строит топ-20 самых частых тем и даёт рекомендации по каким темам писать инструкции в первую очередь.

**yandex-tracker-issue-creator** — создаёт задачи в Yandex Tracker в диалоговом режиме. Уточняет заголовок, описание, очередь и исполнителя, показывает превью и создаёт задачу через API после подтверждения.
