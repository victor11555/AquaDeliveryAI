# Partner Appeals Analyzer
 
Скилл для анализа архива обращений партнёров по продуктам 1С.  
Задача: прочитать все файлы, собрать единую таблицу обращений, найти топ-20 самых частых
тем и дать чёткие рекомендации — по каким темам нужно писать инструкции в первую очередь.

- **Markdown (.md)** — текстовые документы с обращениями в любом формате

## Шаг 1 — Чтение файлов
 
### Excel (.xlsx)
```python
import openpyxl
import pandas as pd
 
# Читаем все листы
xl = pd.ExcelFile("/mnt/user-data/uploads/file.xlsx")
for sheet in xl.sheet_names:
    df = xl.parse(sheet)
    print(df.columns.tolist())
    print(df.head(3))
```
 
**Автоматически определяй колонки** — названия могут отличаться. Маппинг:
 
| Нужное поле | Возможные названия в файле |
|---|---|
| `date` | Дата, Дата обращения, Date, Создано |
| `partner` | Партнёр, Название партнёра, Организация, Компания |
| `category` | Категория, Тема, Тип обращения, Раздел |
| `text` | Текст обращения, Описание, Вопрос, Суть проблемы, Проблема |
| `status` | Статус, Результат, Решение, Ответ |
| `responsible` | Ответственный, Кто ответил, Менеджер |
 
Если колонка не найдена — оставляй поле пустым, не падай с ошибкой.
 
### Markdown (.md)
Читай через bash:
```bash
cat /mnt/user-data/uploads/file.md
```
 
Ищи паттерны:
- Таблицы в формате `| col1 | col2 |` — парси как таблицу
- Блоки вида `## Обращение №123` или `---` — разделители записей
- Списки вида `- Дата: ... Тема: ... Текст: ...`
- Свободный текст — обрабатывай как один блок, извлекай обращения по смыслу

## Шаг 2 — Нормализация записей
 
Для каждого обращения из всех файлов формируй единую запись:
 
```python
{
    "id": порядковый номер (сквозной по всем файлам),
    "source_file": имя исходного файла,
    "date": дата обращения (строка, как есть),
    "partner": название партнёра (если есть),
    "category_original": исходная категория из файла (если есть),
    "text": текст обращения,
    "status": статус/результат (если есть),
    "problem_summary": краткое описание проблемы своими словами — 1 предложение,
    "category_normalized": нормализованная категория (см. Шаг 3),
    "topic_group": конкретная тема для группировки (см. Шаг 3)
}
```
 
**Правила нормализации:**
- Пустые строки, заголовки, служебные записи — пропускай
- Дублирующиеся записи (одинаковый текст + дата + партнёр) — помечай, не удаляй
- Если текст обращения пустой, но есть тема/категория — используй её как текст

## Шаг 3 — Категоризация и группировка по темам
 
### 3.1 Нормализованные категории для 1С
 
Если в исходном файле категория уже есть — уточни и нормализуй её по таблице ниже.  
Если категории нет — определи по тексту обращения.
 
| Категория | Ключевые слова и признаки |
|---|---|
| **Ошибки и сбои программы** | ошибка, вылетает, зависает, не открывается, не запускается, слетела, слетело |
| **Проведение документов** | провести, не проводится, отменить проведение, документ завис, блокировка |
| **Отчёты и печатные формы** | отчёт, печатная форма, не печатается, не выгружается, формирование отчёта |
| **Права доступа и пароли** | нет доступа, не пускает, пароль, роль, пользователь, права |
| **Обмен и синхронизация** | обмен, синхронизация, выгрузка/загрузка данных, не загружается, интеграция |
| **Настройки и конфигурация** | настроить, параметр, константа, учётная политика, конфигуратор |
| **Оплата, касса и банк** | оплата, касса, банк, эквайринг, платёжное поручение, выписка |
| **Справочники** | контрагент, номенклатура, товар, склад, сотрудник, организация |
| **Регламентированная отчётность** | 1С-Отчётность, НДС, декларация, ФНС, ПФР, СФР, ЕФС |
| **Лицензии и обновления** | лицензия, ключ, обновление, релиз, нет ключа, истекла |
| **Зарплата и кадры** | зарплата, начисление, сотрудник, отпуск, больничный, кадры, НДФЛ |
| **Работа с остатками** | остатки, инвентаризация, пересчёт, расхождение, минус |
| **Взаиморасчёты и долги** | долг, задолженность, акт сверки, взаиморасчёты, оплачено/не оплачено |
| **Обучение и консультация** | как сделать, как провести, объясните, покажите, инструкция |
| **Прочее** | всё остальное |
 
### 3.2 Группировка по конкретным темам
 
Внутри каждой категории выделяй **конкретные темы** (topic_group) — группы похожих
обращений об одном и том же действии или проблеме.
 
**Правило:** тема = короткое название (3–7 слов), отражающее суть проблемы.
 
**Примеры:**
- "Ошибка при проведении реализации"
- "Не формируется счёт-фактура"
- "Как закрыть месяц"
- "Не работает обмен с сайтом"
- "Восстановление пароля пользователя"
- "Как настроить учётную политику"
Похожие формулировки **обязательно объединяй** в одну тему, даже если слова разные.

## Шаг 4 — Подсчёт частоты и построение топ-20
 
```python
# Группировка по темам
topic_counts = df.groupby(['category_normalized', 'topic_group']).size().reset_index(name='count')
topic_counts['pct'] = (topic_counts['count'] / len(df) * 100).round(1)
topic_counts = topic_counts.sort_values('count', ascending=False).head(20)
```
 
Для каждой темы из топ-20 фиксируй:
- Порядковый номер в рейтинге
- Категория
- Название темы
- Количество обращений
- % от всех обращений
- Приоритет инструкции (см. ниже)
- Типичный пример обращения (1-2 реальных примера из файлов)
- Типичное решение (если повторяется в статусах/ответах)
- Нужна ли инструкция (да / уже есть / не требуется)
**Приоритет инструкции:**
- 🔴 **Высокий** — >8% обращений, или тема встречается у многих разных партнёров
- 🟡 **Средний** — 3–8%
- ⚪ **Низкий** — <3%

**Форматирование таблицы:**

- Ширина колонок: № (5), Файл (20), Дата (12), Партнёр (25),

```python
from openpyxl import load_workbook
from openpyxl.styles import PatternFill, Font, Alignment
from openpyxl.utils import get_column_letter
 
wb = load_workbook('/mnt/user-data/outputs/appeals-unified.xlsx')
ws = wb.active
 
# Заморозить строку заголовков
ws.freeze_panes = 'A2'
 
# Жирные заголовки
for cell in ws[1]:
    cell.font = Font(bold=True)
    cell.alignment = Alignment(wrap_text=True)
 
# Ширина колонок
col_widths = [5, 20, 12, 25, 25, 30, 40, 50, 25]
for i, width in enumerate(col_widths, 1):
    ws.column_dimensions[get_column_letter(i)].width = width
 
# Подсветить нерешённые
yellow = PatternFill(fill_type='solid', fgColor='FFF9C4')
status_col = 9  # колонка Статус/Решение
for row in ws.iter_rows(min_row=2):
    status = str(row[status_col - 1].value or '').lower()
    if any(w in status for w in ['не решено', 'без решения', 'открыто', 'нет ответа']):
        for cell in row:
            cell.fill = yellow
 
wb.save('/mnt/user-data/outputs/appeals-unified.xlsx')
```

## Обработка ошибок и edge cases
 
| Ситуация | Действие |
|---|---|
| Файл не открывается | Сообщи имя файла и причину, продолжай с остальными |
| Нет колонки с текстом обращения | Используй все текстовые колонки как текст |
| Категории в файле не совпадают с таблицей | Нормализуй по смыслу, исходную категорию сохрани в `category_original` |
| Очень короткий текст (<10 символов) | Помечай как "Неполные данные", не пропускай |
| Файл пустой или только заголовки | Сообщи пользователю, пропусти |
| Дата в нечитаемом формате | Оставляй как строку, не пытайся конвертировать |
| Одно обращение занимает несколько строк | Объединяй по общему ID или дате+партнёру |

## Советы по качеству анализа
 
- **Объединяй похожие темы** — "Ошибка при проведении" и "Не проводится документ" — это одна тема
- **Не дроби категории** — лучше 15 крупных тем с 10 обращениями, чем 50 мелких с 2-3
- **Цитируй реально** — в типичных вопросах используй настоящие тексты из файлов, не выдуманные
- **Смотри на статусы** — если по теме много "не решено" — это красный флаг, даже если тема редкая
- **Партнёры важны** — если одна тема встречается у 10 разных партнёров, это важнее чем 10 обращений от одного
