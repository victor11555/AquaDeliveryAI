# instruction-sync

Синхронизирует инструкции из пользовательских документов с GitHub-репозиторием `victor11555/aqua-delivery-instructions`. Проверяет, что уже есть в репозитории, диагностирует проблемы (устарело, неполное, отсутствует) и создаёт Pull Request с актуальной версией.

## Как работает

1. Читает документ с инструкцией (любой формат)
2. Ищет в репозитории файлы по той же теме через MCP GitHub
3. Сравнивает содержимое — диагностирует: актуально / устарело / неполное / отсутствует
4. Задаёт уточняющие вопросы о правильности шагов
5. Формирует черновик инструкции и ждёт подтверждения
6. После подтверждения создаёт ветку и открывает Pull Request

## Входные данные

- Файл с инструкцией: `.md`, `.txt`, `.pdf`, `.docx`
- Контекст от пользователя: «добавь в репо», «обнови инструкцию», «создай PR»
- Уточнения по процессу в диалоге

## Выходные данные

- Pull Request в `victor11555/aqua-delivery-instructions` с новым/обновлённым `.md` файлом
- Ссылка на PR в ответе
- Описание PR: что изменено, почему, отметка о согласовании

## Триггеры

Загрузка файла + слова «репозиторий», «GitHub», «инструкция»; фразы «создать PR», «обновить репо», «залить инструкцию», «добавь в базу знаний».

## Файловая структура

```
instruction-sync/
├── instruction-sync.md   ← основной файл скилла (алгоритм работы)
└── README.md             ← этот файл
```
