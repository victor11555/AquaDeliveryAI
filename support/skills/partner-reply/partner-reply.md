# Скилл: Заметки оператора → Ответ партнеру (Aqua Delivery TOV)
 
## Что делает скилл
 
Оператор пишет сырые мысли: «ошибка в логах, фиксим, примерно час» — скилл превращает это в готовое сообщение партнеру по стандартам TOV Aqua Delivery.
 
---
 
## Голос Aqua Delivery: 5 принципов
 
Перед генерацией ответа убедись, что он соответствует всем пяти:
 
1. **Конкретика** — что случилось, что делаем, когда ждать. Никаких «разберемся и вернемся».
2. **Дружелюбно, не панибратски** — как уважаемый коллега. Без «!!!» и 😊😊😊. Обращение на «вы».
3. **Уверенно, не снисходительно** — объясняем просто, без технического жаргона без перевода. Ошибки признаем прямо.
4. **Берем ответственность** — даже если причина на стороне партнера или третьего сервиса, ведем к решению.
5. **Доводим до конца** — партнер знает: проблема решена / что будет дальше / куда писать если повторится.

## Алгоритм генерации ответа
 
### Шаг 1: Определи тип ситуации
 
| Тип | Признаки в заметках |
|-----|-------------------|
| 🔍 Первый ответ (проблема не решена) | «смотрю», «ещё не знаем», «нужно проверить» |
| 🔧 Уходит в разработку | «баг», «передаем девам», «срок неизвестен» |
| ↗️ Проблема на стороне партнера/третьего сервиса | «их сторона», «провайдер», «банк», «не наша зона» |
| 🔥 Партнер злится | «негатив», «скандал», «давно мучается» |
| ✅ Решено | «починили», «работает», «закрываем» |
| 📢 Массовый сбой | «у всех», «массово», «проактивно пишем» |
| 💡 Запрос фичи | «нет такой функции», «запрос на доработку» |
 
### Шаг 2: Извлеки ключевые факты из заметок
 
- **Имя партнера** (если есть — используй обязательно, в первом сообщении)
- **Суть проблемы** — что именно сломано/не работает
- **Текущий статус** — что уже делается
- **Тайминг** — когда ждать ответа или решения
- **Следующий шаг** — для партнера или для оператора
- **Обходной путь** — если есть
### Шаг 3: Сформируй структуру по типу
 
**🔍 Первый ответ:**
```
[Имя], [подтверждение что видишь проблему — конкретно].
[Что уже делаешь].
[Тайминг ответа]. [Если будет раньше — напишу сразу.]
```
 
**🔧 Уходит в разработку:**
```
[Имя], [что нашли — конкретно].
Передал разработчикам, задача уже в работе.
[Тайминг или "точный срок пока назвать не могу"].
[Промежуточный апдейт когда].
[Обходной путь, если есть].
```
 
**↗️ Не наша зона:**
```
[Имя], проверил с нашей стороны — [что в порядке].
[Где проблема — конкретно, без технического жаргона].
Вот что нужно сделать:
1. [шаг]
2. [шаг]
[Если что-то пойдет не так — напишите, помогу.]
```
 
**🔥 Негатив:**
```
[Имя], понимаю раздражение — [признание проблемы].
Давайте разберемся: [конкретный уточняющий вопрос].
Я лично займусь этим [тайминг] и дам внятный ответ.
```
 
**✅ Закрытие:**
```
[Имя], [проблема] — починили.
[Что сделай чтобы проверить на своей стороне].
Что было: [краткая причина].
[Что сделали чтобы не повторилось — если применимо].
Если что-то снова пойдет не так — пишите [куда]. Я на связи до [время].
```
 
**📢 Массовый сбой:**
```
⚠️ [Что произошло, с какого по какое время].
Что было: [причина по-человечески].
Сейчас [статус].
Если у вас [конкретное последствие] — напишите, [что сделаем].
```
 
**💡 Запрос фичи:**
```
[Имя], [такой функции нет — по-человечески].
Записал и передам команде продукта.
Пока можно сделать так: [обходной путь если есть].
[Предложение помочь].
```

## Правила стиля (обязательно проверь перед отправкой)
 
### ✅ Обязательно
- Имя партнера в первом сообщении
- Конкретный тайминг (не «скоро», а «через 30 минут», «до 17:00», «в пятницу»)
- Сообщение от первого лица: «я», не «наша команда» и не «специалист»
- Технические термины — с переводом или заменой на понятный язык
- Активный залог: «мы ошиблись», не «ошибка была допущена»
### ❌ Запрещено
- «Добрый день! Проверим ситуацию и вернемся» → нет конкретики
- «Данная функциональность недоступна по техническим причинам» → канцеляризм
- «Обратитесь в техподдержку вашего банка» → перекладываем, не ведем
- «Ваше обращение зарегистрировано под номером #482» → партнеру не важен номер
- «Ждите ответа» → без дедлайна = тревога
- 😊😊😊, !!!!! → несерьезно
- «Ошибка 502 Bad Gateway при таймауте upstream» → без перевода бесполезно
### Emoji: только по назначению
| ✅ Можно | ❌ Нельзя |
|---------|----------|
| ⚠️ — предупреждение о сбое | 😊 — дружелюбие |
| ✅ — подтверждение решения | 😢 — при извинениях |
| 💡 — подсказка/лайфхак | Emoji при сложных инцидентах |
| 📎 — скриншот/ссылка | Emoji в каждой строке |
 
### Длина
- Оптимально: 3–6 строк
- Сложная тема: сначала главное, потом детали
- Шаги — нумерованным списком, не через запятую

## Примеры трансформации
 
### Пример 1
**Заметки оператора:** `синхронизация упала, смотрю логи, ~40 мин, партнер Михаил`
 
**Готовый ответ:**
> Михаил, вижу — заказы не синхронизируются с приложением для водителей. Уже смотрю логи, что-то не то с очередью событий.
>
> Дам ответ в течение 40 минут. Если разберусь раньше — напишу сразу.
 
---
 
### Пример 2
**Заметки оператора:** `баг с фильтром дат в отчете воспроизвели, передали девам, срок неизв, предложить выгружать по месяцам отдельно, партнер Наталья`
 
**Готовый ответ:**
> Наталья, разобрались с фильтром по датам — воспроизвели баг при переходе отчета через месяц. Передал разработчикам, задача уже в работе.
>
> Точный срок пока назвать не могу — зависит от объема правок. Напишу в пятницу: либо скажу дату, либо дам промежуточный статус.
>
> Пока идет исправление: выгружайте отчет за каждый месяц отдельно — так работает корректно.
 
---
 
### Пример 3
**Заметки оператора:** `у нас все норм, проблема на стороне тинькофф - новый апи ключ нужен, дать инструкцию, партнер Дмитрий`
 
**Готовый ответ:**
> Дмитрий, проверил с нашей стороны — всё работает штатно.
>
> Похоже, проблема с онлайн-оплатой на стороне Тинькофф: они обновили интеграцию и теперь нужен новый ключ.
>
> Вот что нужно сделать:
> 1. Войдите в личный кабинет Тинькофф Бизнес → «Интернет-эквайринг»
> 2. Сгенерируйте новый ключ API
> 3. Вставьте его в Aqua Delivery: Настройки → Оплата → Ключ API
>
> Если что-то пойдет не так при настройке — напишите, помогу пройти шаги.

## Чеклист перед отправкой финального сообщения (для закрытия тикета)
 
- [ ] Проблема названа явно
- [ ] Решение подтверждено («проверил, работает», не «должно работать»)
- [ ] Причина объяснена кратко, по-человечески
- [ ] Что сделано чтобы не повторилось (если применимо)
- [ ] Партнеру предложено проверить на своей стороне
- [ ] Обозначено куда писать если что-то пойдет не так
- [ ] Нет подвешенных вопросов — диалог завершен
