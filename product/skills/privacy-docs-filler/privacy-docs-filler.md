# Заполнение юридических документов (Политика + Согласие)

Скилл берёт старые PDF-документы организации, извлекает реквизиты и заполняет
ими новые шаблоны — на выходе **HTML-файлы** (открываются в браузере, поддерживают
печать через Ctrl+P). Шаблоны .docx находятся в `templates/` этого скилла и
используются как промежуточный формат.

---

## Шаблоны

```
<SKILL_DIR>/templates/Политика конфиденциальности.docx
<SKILL_DIR>/templates/Согласие на обработку персональных данных.docx
<SKILL_DIR>/templates/Пользовательское соглашение о порядке покупки товаров.docx
```

Путь к директории скилла — папка, где лежит этот файл.

---

## Процесс (выполнять строго по шагам)

### Шаг 1. Извлечь реквизиты из PDF

Прочитай оба загруженных PDF и найди следующие поля:

| Поле | Описание |
|------|----------|
| `org_type` | «ИП» или «ООО» |
| `org_name` | Название БЕЗ типа. Для ИП — ФИО в **именительном падеже**: «Чугунова Наталья Владимировна». Для ООО — краткое название: «Артезианский источник-Сервис» |
| `inn` | ИНН |
| `kpp` | КПП (только для ООО, иначе null) |
| `ogrn` | ОГРНИП для ИП / ОГРН для ООО |
| `address` | Юридический адрес (он же почтовый) |
| `phone` | Телефон(ы) |
| `email` | Email для обращений |
| `website` | Сайт БЕЗ https:// (например: `palmiry.veel.shop`) |
| `director` | ФИО руководителя. Для ИП совпадает с `org_name` |

Сохрани в файл `requisites.json` в рабочую директорию:

```json
{
    "org_type": "ИП",
    "org_name": "Чугунова Наталья Владимировна",
    "inn": "230812854628",
    "kpp": null,
    "ogrn": "319237500260316",
    "address": "350000, Краснодарский край, г. Краснодар, ул. им. Буденного, д. 129",
    "phone": "89892333302",
    "email": "s89892333302@gmail.com",
    "website": "palmiry.veel.shop",
    "director": "Чугунова Наталья Владимировна"
}
```

### Шаг 2. Запустить скрипт заполнения (.docx)

Определи `SKILL_DIR` как абсолютный путь к папке этого скилла (там, где лежит этот файл).

```bash
python3 "$SKILL_DIR/scripts/fill_docs.py" \
    /path/to/requisites.json \
    /path/to/output_dir \
    "$SKILL_DIR/templates"
```

Скрипт автоматически:
- Убирает все цветовые выделения (yellow, white)
- Проставляет сегодняшнюю дату в оба формата
- Подставляет правильный тип организации везде в тексте
- Для ИП: убирает кавычки вокруг ФИО и склоняет его в дательный падеж там, где нужно
- Заполняет все реквизиты в нужном порядке
- Сохраняет файлы с названием организации в имени файла

**Зависимость:** скрипт использует `pymorphy2` для склонения ФИО.
Если библиотека не установлена, установи перед первым запуском:
```bash
pip install pymorphy2 --break-system-packages
```

### Шаг 3. Конвертировать .docx → HTML и применить стили

После генерации .docx сразу конвертируй оба файла через pandoc, затем применяй стилизацию.
`ORG_LABEL` — название организации как в имени файла (берётся из `org_name` реквизитов).

```bash
# Конвертация через pandoc
for DOCNAME in "Политика конфиденциальности" "Согласие на обработку персональных данных" "Пользовательское соглашение"; do
    pandoc "$OUTPUT_DIR/${DOCNAME} (${ORG_LABEL}).docx" \
        -t html5 --standalone \
        --metadata title="$DOCNAME" \
        -o "$OUTPUT_DIR/${DOCNAME} (${ORG_LABEL}).html"
done

# Добавление CSS-оформления
python3 "$SKILL_DIR/scripts/style_html.py" \
    "$OUTPUT_DIR" \
    "$ORG_LABEL"
```

`style_html.py` добавляет:
- книжный макет с тенью (белая страница на сером фоне)
- шрифт PT Serif, выравнивание по ширине
- стили таблиц с рамками
- поддержку печати (`@media print`)

**Зависимость:** требуется `pandoc` (предустановлен в окружении).

### Шаг 4. Передать файлы пользователю

Отдай пользователю только **HTML-файлы** — они являются финальным результатом.
Ссылки на .docx не нужны (они промежуточные).

## Ключевые нюансы реализации

### ИП vs ООО

| Элемент | ИП | ООО |
|---------|-----|-----|
| Заголовок в политике | «Индивидуальный предприниматель» | «Общество с ограниченной ответственностью» |
| Кавычки вокруг имени | **Нет** — ФИО без кавычек | **Да** — название в «» |
| Поле ОГРН | «ОГРНИП» | «ОГРН» |
| КПП | нет | добавляется в строку ИНН через запятую |
| В согласии (дат. падеж) | «Индивидуальному предпринимателю» | «Обществу с ограниченной ответственностью» |
| ФИО в согласии | **Дательный падеж** (Чугуновой Наталье Владимировне) | Именительный падеж |
| Перед именем в адресе | «ИП» | «ООО» |

### Склонение ФИО ИП

`org_name` всегда хранится в **именительном падеже** (как в паспорте).
Скрипт автоматически определяет пол по отчеству и склоняет ФИО:

- окончания `-овна`, `-евна`, `-вна`, `-чна` → женский род
- окончания `-ович`, `-евич`, `-вич`, `-ич` → мужской род

В согласии (после «Индивидуальному предпринимателю») ФИО подставляется
в дательном падеже. В политике — в именительном.

### Кавычки вокруг ФИО ИП

Шаблоны оформлены под ООО и содержат кавычки `«...»` вокруг плейсхолдера имени.
Для ИП скрипт убирает их в трёх местах:

- **Политика:** убирает `«` перед именем и `»,` после
- **Согласие, начало:** убирает ` «` перед именем и `»` перед `(ИНН:`
- **Согласие, конец:** убирает `«»` в ссылке на политику (ст. 9.2)

### Порядок замен `___________` в политике

В шаблоне политики есть 6 плейсхолдеров `___________` (без запятой) — они
заменяются строго по порядку:

1. Генеральный директор / ФИО ИП
2. Email (п. 2.1)
3. Адрес (п. 2.2)
4. Название ИП/ООО (п. 2.2)
5. Телефон (п. 2.2)
6. URL сайта (п. 3, после `https://`)

Отдельно: плейсхолдер `___________,` (с запятой) — URL согласия в п. 3.

### Email в пп. 11.3 и 11.4

Паттерн ` ______.ru` (с пробелом и .ru) — заменяется на email целиком.

## Пользовательское соглашение — нюансы заполнения

Шаблон рассчитан на ООО, для ИП скрипт автоматически адаптирует:

| Элемент | ООО | ИП |
|---------|-----|----|
| В теле документа (тв.п.) | «Обществом с огр. отв. «Название»» | «Индивидуальным предпринимателем ФИО в тв.п.» |
| Тип орг. в разделе 10 | «Общество с ограниченной ответственностью» | «Индивидуальный предприниматель» |
| Кавычки вокруг названия | «Название», | ФИО, (без кавычек) |
| КПП | заменяется | строка удаляется |
| ОГРН лейбл | «ОГРН:» | «ОГРНИП:» |
| Последняя строка | «Генеральный директор: ФИО» | «Индивидуальный предприниматель ФИО» |

Дополнительные замены (уникальные для этого шаблона):
- `Сыктывкар` → город из поля `address` (извлекается автоматически по шаблону `г. X`)
- `Редакция от 20 мая 2025 года.` → текущая дата в формате `DD месяц YYYY года.`
- Оба URL сайта (`artesian11.ru`, `artesian11.veel.shop`) → `website` из реквизитов

## Что делать, если PDF плохо читается

Если текст PDF нечёткий или реквизиты не видны — попроси пользователя ввести
реквизиты вручную, используя таблицу полей из Шага 1 как чек-лист.
