# privacy-docs-filler

Заполняет юридические документы («Политика конфиденциальности», «Согласие на обработку персональных данных», «Пользовательское соглашение») реквизитами организации. Извлекает реквизиты из старых PDF-документов и подставляет их в шаблоны — на выходе HTML-файлы, готовые к печати.

## Как работает

1. Читает старые PDF-документы партнёра и извлекает реквизиты в `requisites.json`
2. Запускает `scripts/fill_docs.py` — скрипт заполняет `.docx` шаблоны через замену XML-плейсхолдеров
3. Конвертирует `.docx` → HTML через `pandoc`
4. Применяет CSS-оформление через `scripts/style_html.py` (книжный макет, шрифт PT Serif, поддержка печати)
5. Отдаёт HTML-файлы пользователю

Скрипт автоматически определяет тип организации (ИП / ООО) и адаптирует: кавычки вокруг имени, склонение ФИО, ОГРНИП vs ОГРН, наличие КПП.

## Входные данные

- Старые PDF-документы с реквизитами организации
- Или реквизиты вручную: тип (ИП/ООО), название, ИНН, КПП, ОГРН, адрес, телефон, email, сайт, ФИО директора

## Выходные данные

Три HTML-файла (открываются в браузере, печатаются через Ctrl+P):

- `Политика конфиденциальности (Название).html`
- `Согласие на обработку персональных данных (Название).html`
- `Пользовательское соглашение (Название).html`

## Зависимости

```bash
pip install pymorphy2 --break-system-packages   # склонение ФИО
# pandoc — для конвертации docx → html (предустановлен)
```

## Триггеры

Загрузка старых PDF с документами; «заполни документы реквизитами», «создай документы для [организация]», «сделай по этому ИП/ООО».

## Файловая структура

```
privacy-docs-filler/
├── privacy-docs-filler.md         ← основной файл скилла (алгоритм + нюансы ИП/ООО)
├── scripts/
│   ├── fill_docs.py               ← заполнение .docx шаблонов (zipfile + XML замены)
│   └── style_html.py              ← CSS-оформление итоговых HTML-файлов
├── templates/
│   ├── Политика конфиденциальности.docx
│   ├── Согласие на обработку персональных данных.docx
│   └── Пользовательское соглашение о порядке покупки товаров.docx
└── README.md                      ← этот файл
```
