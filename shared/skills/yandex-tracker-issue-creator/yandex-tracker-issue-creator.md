# Yandex Tracker Issue Creator
 
## Обзор
 
Скилл создаёт задачи в Yandex Tracker, собирая все нужные данные у пользователя
до создания. Строгий порядок шагов — обязателен, пропускать нельзя.
 
---
 
## Шаг 1. Уточнить содержание задачи
 
Если пользователь дал только общее описание (не конкретный заголовок и описание),
спроси у него:
 
```
Уточните детали задачи:
 
1. Как примерно назвать задачу? (можете написать черновик — я доработаю)
2. Что важно отразить в описании? (контекст, что нужно сделать, критерий готовности)
```
 
Если пользователь уже дал достаточно информации — этот шаг можно пропустить
и сформировать заголовок и описание самостоятельно на основе его слов.
 
---
 
## Шаг 2. Сформировать заголовок и описание
 
На основе ответов пользователя подготовь:
 
### Правила заголовка (summary)
- Чёткий глагол действия + объект: «Добавить фильтрацию по дате», «Исправить ошибку авторизации»
- Не более 80 символов
- Понятен любому члену команды без контекста
- Без воды: не «Необходимо сделать доработку модуля...», а «Доработать модуль отчётов»
### Правила описания (description)
- 3–7 строк максимум, только суть
- Не пересказывать заголовок
- Использовать маркированный список если задача многошаговая
Шаблон описания:
```
**Задача:** <одно предложение что нужно сделать>
 
**Контекст:** <почему это нужно, если не очевидно>
 
**Критерий готовности:**
- <пункт 1>
- <пункт 2>
```
 
---
 
## Шаг 3. ОБЯЗАТЕЛЬНЫЙ — Выбор очереди
 
**Этот шаг нельзя пропустить.**
 
Используй `queues_get_all` для получения ПОЛНОГО списка очередей:
 
```
yandex-tracker:queues_get_all
```
 
Если очередей много (более одной страницы) — запрашивай постранично до конца,
чтобы показать пользователю полный список. Не обрезай список.
 
Выведи пользователю пронумерованный список:
 
```
В какую очередь создать задачу?
 
1. Название очереди (ключ: QUEUE_KEY)
2. Другая очередь (ключ: KEY2)
...
 
Введите номер очереди:
```
 
Жди ответа. Без выбора очереди — создание задачи недопустимо.
 
---
 
## Шаг 4. ОБЯЗАТЕЛЬНЫЙ — Выбор исполнителя
 
**Этот шаг нельзя пропустить.**
 
Используй `users_get_all` для получения списка пользователей организации:
 
```
yandex-tracker:users_get_all
```
 
Если пользователей много — запрашивай постранично. Показывай имя и логин.
 
Выведи пронумерованный список:
 
```
Кого назначить исполнителем?
 
1. Иван Иванов (login: i.ivanov)
2. Мария Петрова (login: m.petrova)
...
0. Не назначать исполнителя
 
Введите номер:
```
 
Жди ответа пользователя.
 
---
 
## Шаг 5. Показать превью и подтвердить
 
Покажи итоговую задачу перед созданием:
 
```
Создам задачу в очереди QUEUE_KEY:
 
📌 Заголовок: <заголовок>
📝 Описание:
  <описание>
👤 Исполнитель: <имя или "не назначен">
 
Подтвердите создание? (да / нет / отредактировать)
```
 
---
 
## Шаг 6. Создать задачу
 
После подтверждения:
 
1. Вызови `queue_get_fields` для выбранной очереди — проверь обязательные поля
2. Если есть обязательные поля помимо заголовка — уточни у пользователя
3. Вызови `issue_create`:
   ```
   queue: <ключ очереди>
   summary: <заголовок>
   description: <описание>
   assignee: <логин исполнителя, если выбран>
   ```
 
---
 
## Шаг 7. Отчёт о результате
 
```
✅ Задача создана!
 
QUEUE-XXX — <заголовок>
Очередь: <название> | Исполнитель: <имя или "не назначен">
```
 
---
 
## Обработка ошибок
 
- Очередь не найдена → предложи ввести ключ вручную
- Пользователь не найден в списке → предложи поиск по имени через `users_search`
- Задача не создалась → сообщи об ошибке с текстом из ответа API
- Нет прав на очередь → предложи выбрать другую
