# 1C Implementation — Среда отдела

Два направления работы: **аудит** (анализ встречи → Excel-план) и **первоначальная настройка** (автоматизация через Playwright). Читай этот файл целиком перед стартом.

---

## Направление 1. Аудит партнёра → план + задачи в трекере

Два скилла, которые работают последовательно:

| Скилл | Как вызвать | Что делает |
|-------|-------------|------------|
| `aqua-implementation` | `/aqua-implementation` | Транскрибация встречи → Excel-план + `data.json` |
| `voronka-vnedreniya-tracker` | `/voronka-vnedreniya-tracker` | `data.json` → 40+ задач в Yandex Tracker |

**Скилл `aqua-implementation`:** `/aqua-implementation`

**Триггер:** пользователь кидает транскрибацию встречи с партнёром, просит «составь план», «заполни по транскрибации», «оформи итоги аудита».

**Что на выходе:** один `.xlsx` файл с двумя листами — «Описание процессов» (31 процесс: было/стало/риски) и «План внедрения» (этапы с датами и стоимостью).

**Алгоритм скилла (интерактивный — скилл задаёт вопросы, не пропускай стопы):**

```
0. Определить тип партнёра: Новый (с нуля) или Действующий (уже в системе, нужна только 1С)
   → СТОП, если тип не указан явно — спросить
1. Прочитать references/audit_questions.md — структура аудита
2. Проанализировать транскрибацию, заполнить «Было» по каждому процессу
3а. СТОП — спросить про все процессы с пустым «Было» (список в чат, ждать ответа)
4. Выбрать шаблон плана:
   - Новый → references/plan_template.json (5 этапов, Этапы 0–5)
   - Действующий → references/plan_template_existing.json (13 шагов по 1С)
4а. СТОП — запросить размер компании (малая/средняя/большая), потом трудоёмкость каждого шага
    → Расчёт дат: макс 4 часа в день, только рабочие дни
5. Собрать data.json
6. pip install openpyxl; python3 scripts/build_excel.py data.json <путь>.xlsx
7. Показать ссылку + сводку (сколько процессов заполнено)
```

**Что адаптируется в плане под партнёра:**
- Нет 1С → убрать перенос из старой базы
- Есть IP-телефония → добавить шаг в Этап 4
- Нет маркировки → убрать шаги ЧЗ/GTIN
- Два юрлица через одного водителя → добавить «Схема отложенного пробития чека»

## Направление 2. Первоначальная настройка нового партнёра

Все агенты работают через **Playwright MCP** (`@playwright/mcp --isolated`). Конфигурация: `.mcp.json` в корне репозитория.

### Стандартный порядок (3 агента строго по порядку)

```
1. setup-internet-shop        → кабинет admin + 1С: интеграция интернет-магазина
2. disable-1c-background-jobs → 1С: отключение фоновых и регламентных заданий
3. 1c-history-settings        → 1С: версионирование 11 объектов
```

Запускать через **оркестратор** (`orchestrator`) — он сам соберёт данные и запустит всё по порядку. Или каждый агент вручную если нужен конкретный шаг.

**Триггеры оркестратора:** «новый партнёр», «настрой партнёра», «подключи партнёра», «полная настройка».

---

### Агент 1: setup-internet-shop

**Что делает:** Шаги 1-8 настройки интеграции.

| Шаг | Где | Что |
|-----|-----|-----|
| 1 | Кабинет admin `/exchange/settings/update/` | Включить API обмен + обмен с 1С приложением |
| 2 | Кабинет admin `/exchange/account/index/` | Создать аккаунт «1C Aqua Delivery» (email: `{ДОМЕН}.1c@mail.ru`, пароль: 16 символов) |
| 3 | 1С | Войти под Администратором |
| 4 | 1С → CRM → Интернет-магазин | Открыть форму интеграции |
| 5 | 1С → Авторизация | URL заказов + логин/пароль аккаунта → «Проверить соединение» |
| 6 | 1С → Статусы | 6 статусов (new/approved/performed/completed/undelivered/removed), по умолчанию «Новый» |
| 7 | 1С → ДополнительныйФункционал | 5 опций: маршрутные листы, расходные накладные (авто + обновление), чек ККМ, дата отгрузки |
| 8 | 1С → Заказы → Загрузка | Организация = `{ORG_NAME}` |

**Данные для запуска:**
- `URL_1C`, `LOGIN_1C`, `PASS_1C`
- `URL_ADMIN`, `URL_ORDERS`, `ADMIN_EMAIL`, `ADMIN_PASS`
- `ORG_NAME`, `DOMAIN`

---

### Агент 2: disable-1c-background-jobs

**Что делает:** Отключает всё лишнее в 1С.

| Раздел | Что отключается |
|--------|----------------|
| Интернет-поддержка и сервисы | Классификаторы (авто), адреса через интернет, Новости, Обновление версии → «Отключена», Проверка контрагентов, 1СПАРК Риски, Центр мониторинга → «Запретить отправлять», Внешние компоненты → «Отключено» |
| Общие настройки | Полнотекстовый поиск данных |
| Регламентные задания | 13 заданий (см. список ниже) |

**13 регламентных заданий для отключения:**
1. 1С:Номенклатура. Выгрузка номенклатуры в электронные каталоги
2. Загрузка курсов валют
3. Заполнение контрагентов на мониторинге СПАРК Риски
4. Обновление внешних компонент *(может отсутствовать)*
5. Обновление индексов СПАРК Риски
6. Обновление настроек участников СБП *(может отсутствовать)*
7. Обновление регламентированных отчетов
8. Обновление факторов рисков СПАРК
9. Очистка замеров времени *(может отсутствовать)*
10. Получение уведомлений Монитора Портала 1С:ИТС
11. Проверка ведения учета
12. Синхронизация рекомендаций 1С:Плюс
13. Синхронизация с 1С:Плюс

**Данные для запуска:** `URL_1C`, логин, пароль.

### Агент 3: 1c-history-settings

**Что делает:** Ставит «При записи» + «Бессрочно» для 11 объектов.

**11 объектов:**

Путь: Настройки → Общие настройки → История изменений → Настроить.

**Данные для запуска:** `URL_1C`, логин, пароль.

### Дополнительный агент: setup-kafka-exchange

**Триггеры:** «настрой kafka», «событийный обмен с CRM», «обмен с kafka 1С».

**Что делает:** Aqua Delivery → Событийный обмен с CRM → создаёт запись обмена, настраивает брокер Kafka (ydb Yandex Cloud), выгрузку цен (2 топика), пользователя «Обмен с Kafka», регламентное задание 120 сек.

**Данные:** `URL_1C`, логин, пароль, домен партнёра, пароль для пользователя Kafka.

**Критический порядок заполнения брокера:** сначала выбрать протокол (sasl_ssl) и механизм (PLAIN), ПОТОМ заполнять текстовые поля — иначе поля сбрасываются.

---

### Дополнительный агент: setup-1c-user

**Триггеры:** «создай пользователя 1С», «создание пользователя», «новый пользователь 1С УНФ», «добавь пользователя партнёра», «доделай пользователя в 1С».

**Что делает:** Создаёт или доделывает пользователей в 1С УНФ (Настройки → Настройки пользователей и прав → Пользователи). Для каждого ФИО из списка:
- Если пользователь уже частично заполнен — доделывает (без пароля)
- Если не существует — создаёт с нуля (Полное имя → Tab → автозаполнение «Имя для входа»)
- В обоих случаях включает 3 чекбокса на «Настройки входа» (Аутентификация 1С:Предприятия, Потребовать установку пароля при входе, Показывать в списке выбора) и добавляет в группу доступа «Администратор (партнер)»
- **НИКОГДА** не устанавливает и не генерирует пароль вручную

**Данные:** `URL_1C`, логин/пароль администратора, список ФИО для создания/доделывания.

### Мета-агент: agent-builder

**Триггеры:** «выполни инструкцию», «сделай агента», «новая инструкция», «автоматизируй шаги».

**Принцип работы:**
1. Получает инструкцию + данные для входа
2. **Сам выполняет** все шаги в браузере через Playwright — ведёт лог каждого действия
3. **Только после реального выполнения** создаёт файл агента на основе того, что реально сработало

Файлы новых агентов сохраняются в `skills/initial-setup/<agent-slug>/`.

## Технические особенности веб-клиента 1С

### Главные правила

| Проблема | Решение |
|----------|---------|
| ID форм динамические (`form4_`, `form5_`...) | Всегда искать через `querySelector` / `page.evaluate()` |
| Таблицы виртуализированы | Скролл только через `ArrowDown`, `Home`, `End` — не `mouse.wheel` |
| Скролл панелей | `element.scrollTop += N` — не `page.mouse.wheel` |
| `fill()` не регистрируется | Использовать `pressSequentially()` в полях 1С (Наименование, пароли, топики) |
| «Все лицензии заняты» | Нажать **Перезапустить**, подтвердить `beforeunload`, войти повторно |
| «Не удалось начать редактирование» | Нажать **Перехватить редактирование** |
| Непредвиденная ошибка | Нажать **Перезапустить**, подождать 8 сек, повторить вход |

### Состояния переключателей

```
switch zoomI select  → включён  (кликнуть чтобы выключить)
switch zoomI         → выключен ✅
radio zoomI select   → выбрана эта радиокнопка
checkbox zoomI       → чекбокс выключен (без select)
```

### Полезные селекторы

| Элемент | Selector |
|---------|---------|
| Меню «Настройки» в сайдбаре | `#themesCell_theme_7` |
| Общие настройки | `#cmd_0_11_txt` |
| Таблица регламентных заданий | `#form8_ТаблицаРегламентныеЗадания` |
| Скроллируемая панель интернет-поддержки | `#form5_$scrl` |
| Поиск в таблице истории | `input[id*="СтрокаПоиска"]` |

### Производительность

Snapshot делать только когда нужен ref для клика. Screenshot — только для ключевых этапов. Не злоупотреблять — тормозит выполнение.

## Контекст продукта Aqua Delivery

Общие справочники по продуктам компании — [`context/`](../context/), доступны всем отделам. Открывай нужный файл, когда для задачи требуется понимание интерфейса, бизнес-логики или архитектуры конкретного продукта.

| Файл | Что внутри |
|------|-----------|
| `AquaDelivery_ProductContext_AI.md` | Полный контекст: бизнес-модель, продуктовая карта, БД, интеграции, роли, ограничения и техдолг |
| `crm_ai.md` | CRM «Кабинет заказов» — разделы, формы, поля, маршрутные листы |
| `admin_panel_ai.md` | Административная панель — товары, пользователи, продажи, лояльность, маршрутизация, обмен |
| `client_app_ai.md` | Клиентское приложение (PWA/моб.) — каталог, корзина, заказы, бонусы, профиль |
| `courier_app_ai.md` | Курьерское приложение — заказы, маршруты, отчёт за день |
