# aqua-implementation

Генерирует Excel-файл с планом внедрения Aqua Delivery (1С УНФ) на основе транскрибации аудиторской встречи с партнёром. Анализирует текст встречи, извлекает текущие бизнес-процессы партнёра и формирует структурированный документ.

## Как работает

1. Определяет тип партнёра (новый / действующий) — от этого зависит шаблон плана
2. Читает вопросы аудита из `references/audit_questions.md`
3. Анализирует транскрибацию и заполняет 31 бизнес-процесс («Было», «Стало», «Риски»)
4. Запрашивает уточнения по пустым процессам у пользователя
5. Формирует план внедрения из нужного шаблона (`plan_template.json` или `plan_template_existing.json`)
6. Уточняет сроки и трудоёмкость задач у пользователя
7. Генерирует `.xlsx` файл через `scripts/build_excel.py`

**Скилл интерактивный** — в процессе задаёт уточняющие вопросы и ждёт ответов перед продолжением.

## Входные данные

- Транскрибация аудиторской встречи (сырой текст)
- Тип партнёра: новый (запуск с нуля) или действующий (уже в системе, нужна только 1С)
- Опционально: название компании, дата запуска, кол-во клиентов, нужные модули

## Выходные данные

Один `.xlsx` файл с двумя листами:

| Лист | Содержание |
|------|-----------|
| Описание процессов | 31 бизнес-процесс: «Было», «Стало», «Риски/комментарии» |
| План внедрения | Новый партнёр → Этапы 0–5; Действующий → 13 шагов по 1С |

## Триггеры

Отправка транскрибации встречи; «составь план внедрения», «заполни план по транскрибации», «сделай документ по итогам аудита».

## Файловая структура

```
aqua-implementation/
├── aqua-implementation.md          ← основной файл скилла (алгоритм)
├── references/
│   ├── audit_questions.md          ← 7 блоков вопросов для аудита
│   ├── processes.json              ← 31 бизнес-процесс со стандартными описаниями
│   ├── plan_template.json          ← шаблон плана для нового партнёра (Этапы 0–5)
│   └── plan_template_existing.json ← шаблон плана для действующего партнёра (13 шагов)
├── scripts/
│   └── build_excel.py              ← генерация .xlsx (openpyxl)
└── README.md                       ← этот файл
```
