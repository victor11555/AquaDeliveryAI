# Создание / доделка пользователей в 1С УНФ

Агент принимает список ФИО пользователей. Для каждого ФИО:
1. Проверяет, существует ли уже пользователь с таким полным именем (хотя бы частично заполненный).
2. Если существует — открывает запись и доделывает (без установки пароля).
3. Если не существует — создаёт нового пользователя с нуля.
4. В обоих случаях добавляет пользователя в группу доступа "Администратор (партнер)".
5. НИКОГДА не устанавливает и не генерирует пароль вручную — поле "Пустой пароль" /
   кнопку "Установить пароль..." не трогать. Галочка "Потребовать установку пароля при
   входе" обеспечивает, что пользователь сам задаст пароль при первом входе.

## Шаг 0. Данные для входа

Уточни у пользователя (если не указано):
- URL веб-клиента 1С: `https://server-1c.service.appsol.ru/{ДОМЕН}/ru_RU/`
- Логин: обычно "Администратор"
- Пароль администратора
- Список ФИО для создания/доделывания (например: "Голенков Алексей Сергеевич", "Козловский Михаил")

## Шаг 1. Войти в систему и открыть список пользователей

```
mcp__playwright__browser_navigate → URL_1С
```

Если при навигации/загрузке всплывают диалоги `beforeunload` (зависшие от прошлой
сессии) — подтверждай через `browser_handle_dialog` с `accept: true`.

Перейти: **Настройки → Настройки пользователей и прав → Пользователи**.

Список пользователей сгруппирован по "Группы доступа" с заголовком колонки "Пользователь ↑/↓".
Чтобы увидеть ВСЕХ пользователей и проверить наличие нужного ФИО — кликни по заголовку
колонки "Пользователь" (сортировка), список покажет все записи (обычно их немного:
Администратор, Обмен с Kafka, ранее созданные пользователи).

Поиск по ФИО через текстовое поле "Поиск (Ctrl+F)" в списке может не фильтровать
группированный список надёжно — лучше визуально проверить полный (отсортированный)
список из 3-10 записей.

## Шаг 2A. Если пользователь УЖЕ СУЩЕСТВУЕТ (частично заполнен)

Открой запись двойным кликом / кликом по строке. Форма откроется с динамическим id
(`form7_`, `form11_`...).

1. Проверь вкладку **"Основное"** → подвкладку **"Настройки входа"**:
   - "Имя (для входа):" — должно быть заполнено (например "ГоленковА")
   - "Аутентификация 1С:Предприятия" — checkbox должен быть включён
   - "Потребовать установку пароля при входе" — checkbox должен быть включён
   - "Показывать в списке выбора" — checkbox должен быть включён
   - Если что-то не включено — включи (см. Шаг 4 про работу с чекбоксами)
   - **Пароль НЕ трогать.**

2. Перейди на вкладку **"Права доступа"** (клик по заголовку вкладки, ref вида
   `generic [ref=eXXXX]: Права доступа`).

3. На подвкладке "Группы доступа" нажми **"Включить в группу"**.

4. В открывшемся дереве "Дополнительные группы" найди и выбери "Администратор (партнер)":
   - Введи в поле поиска дерева текст `Администратор (партнер)` (через `pressSequentially`,
     slowly: true)
   - Кликни по найденной строке "Администратор (партнер) (Дополнительные группы)"
   - Нажми **"Выбрать"**

5. Вернись на вкладку **"Основное"** → нажми **"Записать и закрыть"**.

## Шаг 2B. Если пользователь НЕ СУЩЕСТВУЕТ — создать с нуля

1. В списке "Пользователи" нажми **"Создать"**.
2. В поле "Полное имя:" введи ФИО (например "Козловский Михаил") через `pressSequentially`,
   затем нажми Tab — система автозаполнит "Имя (для входа):" по паттерну
   Фамилия+ПерваяБукваИмени (например "КозловскийМ").
3. Перейди на подвкладку **"Настройки входа"**.
4. Проверь и при необходимости включи чекбоксы (см. Шаг 4):
   - "Аутентификация 1С:Предприятия" — обычно уже включена по умолчанию
   - "Потребовать установку пароля при входе" — обычно ВЫКЛЮЧЕНА по умолчанию, нужно включить
   - "Показывать в списке выбора" — обычно уже включена по умолчанию
5. Закрой всплывающую подсказку про требования к паролю, если появилась
   (кнопка "Закрыть" с крестиком рядом с текстом подсказки).
6. Нажми **"Записать"** (не "Записать и закрыть" — нужно остаться в форме для добавления группы).
7. Если появится диалог "Включить обсуждения?" — нажми **"Нет"**.
8. Перейди на вкладку **"Права доступа"** → **"Включить в группу"** → найди и выбери
   "Администратор (партнер)" (см. Шаг 2A.4) → **"Выбрать"**.
9. Вернись на **"Основное"** → **"Записать и закрыть"**.

## Шаг 3. Проверка результата

После сохранения список "Пользователи" (отсортированный по "Пользователь") должен
показывать каждого обработанного пользователя в группе "Администратор (партнер)".

## Шаг 4. Работа с чекбоксами (технические детали)

Чекбоксы 1С реализованы как `<div>` с классом `checkbox zoomI` (выключен) или
`checkbox zoomI select` (включён). Простой `.click()` по тексту лейбла не всегда
переключает чекбокс. Надёжный способ — найти DIV чекбокса по id-паттерну и отправить
полную последовательность событий по его центру:

```js
() => {
  // id чекбоксов на вкладке "Настройки входа" (паттерн form{N}_...)
  const ids = [
    'ПользовательИБАутентификацияСтандартная',   // Аутентификация 1С:Предприятия
    'ПотребоватьСменуПароляПриВходе',             // Потребовать установку пароля при входе
    'ПользовательИБПоказыватьВСпискеВыбора'       // Показывать в списке выбора
  ];
  ids.forEach(suffix => {
    const el = document.querySelector(`[id$="${suffix}"][class^="checkbox"]`);
    if (el && !el.className.includes('select')) {
      const rect = el.getBoundingClientRect();
      const x = rect.x + rect.width/2, y = rect.y + rect.height/2;
      const opts = {bubbles: true, cancelable: true, view: window, clientX: x, clientY: y, button: 0};
      el.dispatchEvent(new PointerEvent('pointerdown', opts));
      el.dispatchEvent(new MouseEvent('mousedown', opts));
      el.dispatchEvent(new PointerEvent('pointerup', opts));
      el.dispatchEvent(new MouseEvent('mouseup', opts));
      el.dispatchEvent(new MouseEvent('click', opts));
    }
  });
}
```

Проверка состояния чекбокса:
```js
() => {
  const el = document.getElementById('form12_ПотребоватьСменуПароляПриВходе');
  return el.className; // "checkbox zoomI select" = включён, "checkbox zoomI" = выключен
}
```

## Технические особенности

- ID форм динамические (`form4_`, `form7_`, `form11_`, `form12_`, `form15_`...) —
  всегда искать через `document.querySelector` / `getElementById` с частичным
  совпадением (`[id$="..."]`, `[id^="..."]`), не хардкодить полный id.
- При переключении вкладок ("Основное" ↔ "Права доступа") id внутренних элементов формы
  меняются (например `form7_` → `form9_`) — перезапрашивай элементы заново после
  каждого переключения вкладки.
- Вкладки верхнего уровня формы пользователя кликаются по ref из snapshot (текст вкладки:
  "Основное", "Настройки пользователя", "Права доступа", "Сотрудники пользователя",
  "Настройки").
- Дерево "Дополнительные группы" в диалоге "Включить в группу" поддерживает поиск —
  вводи "Администратор (партнер)" в поле поиска (Ctrl+F) дерева, дождись фильтрации,
  кликни по найденной строке, затем "Выбрать".
- Если "Все лицензии заняты" → "Перезапустить" → подтвердить `beforeunload` через
  `browser_handle_dialog({accept: true})`.
- Если "Не удалось начать редактирование" → "Перехватить редактирование".
- Диалог "Включить обсуждения?" при первом сохранении нового пользователя — отвечать "Нет".
- НЕ устанавливать пароль: не нажимать "Установить пароль...", не трогать поле "Пустой пароль".

## Итоговый чеклист (на каждого пользователя)

- [ ] Полное имя заполнено корректно
- [ ] Имя (для входа) заполнено (автоматически или вручную)
- [ ] Аутентификация 1С:Предприятия — включена
- [ ] Потребовать установку пароля при входе — включена
- [ ] Показывать в списке выбора — включена
- [ ] Пароль НЕ установлен вручную
- [ ] Группа доступа "Администратор (партнер)" добавлена
- [ ] Форма сохранена ("Записать и закрыть")
- [ ] Пользователь виден в общем списке "Пользователи" в группе "Администратор (партнер)"
